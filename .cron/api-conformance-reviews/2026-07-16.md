# v1 API contract-conformance review — 2026-07-16 (initial ledger pass)

*First run of the new product-contract-conformance loop (Loop Library #076, adapted). Read-only.
Reconciles the published OpenAPI contract against the `/api/v1` implementation. No code changed.
Establishes the baseline ledger the recurring runs measure against.*

## TL;DR

- **Baseline gate GREEN.** `src/lib/api/v1/openapi.test.ts` = **15/15 pass**. The contract is
  already **strongly locked at two levels the loop would otherwise have to build**: (a) every
  request-schema's property set is compile-time-pinned to the TS input shapes via
  `Record<keyof EntryInput, true>` (…Sense/Sentence/Patch/Dialect) inventories — TS won't compile if
  a field drifts; (b) the **exact path+method surface** (52 route files) is asserted, plus "every op
  is tagged" and "no `[DRAFT]`/`x-status` leftovers." So the *shape* and *surface* dimensions of
  conformance are effectively regression-proof today.
- **The open dimension is field-level SEMANTICS + ENFORCEMENT-TEST coverage** — what the schema-key
  lock can't see: required-vs-optional enforcement, enum/range validation, permission rules, and
  documented behavior actually holding in the handler. That's the ledger the recurring runs fill in.
- **No confirmed contradiction found this pass** (a light first sweep, not the full per-field
  ledger). One honest coverage metric below to anchor the "weak rows" count.

## Enforcement-test coverage (the honest "weak rows" baseline)

- **52** `/api/v1/**/+server.ts` route files; **18** carry a co-located `server.test.ts`.
- The uncovered majority is dominated by **media routes** (entry/sense/sentence/text
  audio/photo/video) — but those route through the shared `media-route-handlers.ts`, which **does**
  have `media-route-handlers.test.ts` + `validate-media-bytes.test.ts`, so many are covered
  centrally rather than per-route. The **first full ledger run's job** is to classify each uncovered
  route as *covered-by-shared-test* (fine) vs *genuinely unpinned* (a `weak` row worth a regression
  test) — the raw 18/52 is a starting signal, not a verdict.
- Notably thin (no co-located test AND not obviously shared-covered), to classify first:
  `dialects` (POST/GET collection), `orthographies/[code]`, `tags` (collection), the `grammar`
  clause-slots / glossing-abbreviations / section-sentences endpoints, and `sources/[sourceId]`.

## Ledger method for the next (full) run

For each path+method and each request/response field, extract the contract's promise and mark it
`proved` / `weak` / `contradicted` / `unimplemented` against the handler + `*-input.ts` validator +
enforcement test. Rank confirmed mismatches by agent-facing impact (a required field the handler
ignores or a permission the code skips > a missing example > prose). Reconcile **code → spec**;
report genuine spec bugs rather than rewriting the contract.
