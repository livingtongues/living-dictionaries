## 5. Coverage gaps (Phase B)

1. **`remote-log.ts` ‚Äî cap/coalesce repeated identical errors.** *(HIGH ‚Äî grounded in today.)* The P1 logged **8,170 byte-identical rows from a single session**. `remote-log.ts` should emit the first *N* of an identical (message+stack) error, then coalesce to a periodic `"√óN more"` ‚Äî this both **slashes junk volume** and makes a runaway loop *more* legible (a single "√ó8170" row screams "loop"). Path: `site/src/lib/debug/remote-log.ts` error hook ‚Üí dedupe key = `message|stack_head`, counter + flush. Level: unchanged. *Would have made today's incident one clean row instead of 8k.*
2. **Sync engine ‚Äî stop the infinite `schema_outdated` retry + prompt reload.** *(HIGH.)* A `client_behind`/`schema_outdated` **409** means the client bundle is stale; retrying can never succeed until reload, yet Greg's client retried **1,707√ó/day**. On repeated 409-schema_outdated: **hard-cap/exponentially back off** and surface a user-facing "reload to update" toast. Emit **one** `sync_failed` warn then suppress duplicates (ties to #1). Path: the dict sync worker's push/pull error handling.
3. **Entries `?q=` parse ‚Äî coerce + log malformed.** *(MEDIUM.)* A raw `?q=hua` becomes a string and crashes the filter binding (¬ß1 P3). Coerce non-object parsed `q` ‚Üí default object; emit a `warn` (`event:"malformed_q_param"`, `context:{ raw }`) so we can see how often links/bots hit it. Path: entries `+page.ts` search-params parse + `EntryFilters.svelte:160`.
4. **SSR gloss-map guard + context.** *(LOW.)* Guard the `vernacularName` map and attach `{ route, dict_id }` to the server log so the 1/day SSR 500 is actionable.
