### ЁЯФ┤ P1 тАФ WorldMap `getBoundingClientRect` on null, looping every frame
- **~10,850 rows / 24h** (`8,190` on build `тАж104848` + `2,664` on `тАж458430`, both **current** builds тАФ a live regression, not stale tabs). `last_seen` 11:32 UTC.
- **Node 4 = `src/routes/+page.svelte`** (homepage). Confirmed against the minified prod bundle (`Ye @ 4.CvH5oEUf.js:1:154210`) тЖТ `src/lib/components/home-v2/map/WorldMap.svelte:589`: `const rect = container.getBoundingClientRect()` тАФ **unguarded**, unlike the guarded `map_frame?.` in `HeroUnit.svelte:95`.
- **Why it storms:** the highlight-dot **pulse** ripple calls `schedule_draw()` (queues the next frame) *before* execution reaches line 589. So `draw()` re-queues, then throws тАФ forever, as long as a `selected_dict` is set while `container` is null (teardown/hydration ordering, e.g. navigating away mid-pulse). One session logged **6,236** of these; an anonymous visitor **898 in ~90s**.
- **Fix (recommended, NOT applied):** `const rect = container?.getBoundingClientRect(); if (!rect) return` (+ optionally fold `!container` into `draw()`'s top guard so the pulse loop idles harmlessly). Full write-up + verification steps in the issue file.

## 5. Coverage gaps (Phase B)

1. **`remote-log.ts` тАФ cap/coalesce repeated identical errors.** *(HIGH тАФ grounded in today.)* The P1 logged **8,170 byte-identical rows from a single session**. `remote-log.ts` should emit the first *N* of an identical (message+stack) error, then coalesce to a periodic `"├ЧN more"` тАФ this both **slashes junk volume** and makes a runaway loop *more* legible (a single "├Ч8170" row screams "loop"). Path: `site/src/lib/debug/remote-log.ts` error hook тЖТ dedupe key = `message|stack_head`, counter + flush. Level: unchanged. *Would have made today's incident one clean row instead of 8k.*
2. **Sync engine тАФ stop the infinite `schema_outdated` retry + prompt reload.** *(HIGH.)* A `client_behind`/`schema_outdated` **409** means the client bundle is stale; retrying can never succeed until reload, yet Greg's client retried **1,707├Ч/day**. On repeated 409-schema_outdated: **hard-cap/exponentially back off** and surface a user-facing "reload to update" toast. Emit **one** `sync_failed` warn then suppress duplicates (ties to #1). Path: the dict sync worker's push/pull error handling.
3. **Entries `?q=` parse тАФ coerce + log malformed.** *(MEDIUM.)* A raw `?q=hua` becomes a string and crashes the filter binding (┬з1 P3). Coerce non-object parsed `q` тЖТ default object; emit a `warn` (`event:"malformed_q_param"`, `context:{ raw }`) so we can see how often links/bots hit it. Path: entries `+page.ts` search-params parse + `EntryFilters.svelte:160`.
4. **SSR gloss-map guard + context.** *(LOW.)* Guard the `vernacularName` map and attach `{ route, dict_id }` to the server log so the 1/day SSR 500 is actionable.
