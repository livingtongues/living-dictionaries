### 🟡 P3 (new) — waveform ▶ play before wavesurfer initializes

Root cause **verified in current source** — `src/lib/components/audio/Waveform.svelte`:

```svelte
let wavesurfer                     // plain let, NOT $state
onMount(async () => {
  const WaveSurfer = (await import('wavesurfer.js')).default   // async — takes time
  wavesurfer = WaveSurfer.create({ … })
  …
})
function startStop() {
  if (playing) wavesurfer.stop()
  else         wavesurfer.play()   // ← throws if clicked before onMount's import resolves
  playing = !playing
}
```

The play button is live from first paint, but `wavesurfer` is only assigned *after* the dynamic
`import('wavesurfer.js')` resolves. A user who clicks ▶ immediately (the drilled session recorded audio
then hit play right away, breadcrumbs `Presione para grabar → Terminar la grabación → button`) fires
`wavesurfer.play()` on `undefined`. 7 rows / 1 user / 1 session, all on the current build, on
`/guaycura/entries` (Guaycura — a real Baja California language; the MX-BCS geo confirms a genuine
contributor, not the conlang fork). **Long-standing**, not a regression (file last touched in the
Svelte-5 migration).

- **Recommend (do NOT apply):** null-guard the calls and only toggle `playing` when `wavesurfer` exists
  — `wavesurfer?.play()` / `wavesurfer?.stop()`, and early-return from `startStop` if `!wavesurfer`.
  Optionally disable the ▶ button until `onMount` finishes (a `ready = $state(false)` set at the end of
  `onMount`). → action item.
