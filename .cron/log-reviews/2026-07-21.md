### Persistent OPFS finding

Ngemba's production DB contains `sentences.discourse_role` **and** a `migrations` row for `20260714_structured_grammar.sql`. The Android browser's local file has the column but evidently lacks the ledger row, so `ensure_migrations()` runs the `ALTER TABLE` again and fails with `duplicate column name`. `worker/db-client.ts` spends three fast retries, resigns, and re-elects indefinitely; it never replaces the persistently broken file. The recovery issue requires dirty-row preservation before any reset.
