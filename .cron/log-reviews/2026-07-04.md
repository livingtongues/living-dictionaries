### The three new `rhenic` bugs (current 06:07 build)
All from one contributor rapidly editing + recording on `rhenic` this morning (08:00–08:17 UTC):

- **🟠 P2 — media-upload eager-destructure crash.** <File path="site/src/lib/components/audio/upload-audio.ts" line="28" />
  (and `upload-image.ts:30`, `upload-video.ts:28`) do
  `const { data: { presigned_upload_url, … }, error } = await api_upload(…)`. The `_call.ts` contract
  returns `{ data: null, error }` on failure, so destructuring `data.presigned_upload_url` throws
  **before** the `if (error)` guard runs. Breadcrumbs show the user recording 2s clips in a loop; when
  one upload-URL request failed, the record modal threw instead of showing the error — and the crash
  **swallows `error.message`**, so we can't see *why* the request failed. Affects audio, image and
  video uploads identically. → **Phase B #1** (highest-value fix today).
- **🟡 P3 — FK-constraint poison-pill push.** <File path="site/src/lib/db/server/dictionary-sync-helpers.ts" line="93" />
  runs the whole push under `defer_foreign_keys=ON` + one transaction, so a dirty child row (sense/
  sentence/media/junction) whose server parent no longer exists throws `FOREIGN KEY constraint failed`
  at `COMMIT` (line 185) and rolls back the **entire** batch → client 500 → `sync_failed` kind:other →
  retry. One occurrence (rhenic, 08:10:18), **did not loop** (transient create/delete ordering that
  resolved). But the log is bare — no failing `table_name`/`row_id` — so a *permanently* orphaned dirty
  row would silently 500 every push forever with nothing to diagnose. → **Phase B #2.**
- **🟡 P3 — `effect_update_depth_exceeded`.** One `crash` at 08:01 on the rhenic entry-edit page during
  a rapid Save×3 + search burst. A Svelte reactive loop; not reproducible from logs alone. **Watch** —
  if it recurs on entry-edit, trace the offending `$effect`/`$derived` on the entry page.
