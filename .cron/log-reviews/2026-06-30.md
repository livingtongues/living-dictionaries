### 🟠 gloss_languages.map (homepage dictionary card)
- **Where:** <File path="site/src/lib/components/home/SelectedDict.svelte" line="43" /> guards only
  truthiness (`{#if dictionary.gloss_languages}`) then calls `.map(...)`. If the value reaches the
  component as an **unparsed JSON string** (e.g. a freshly-created dict's row before re-hydration), a
  string is truthy but has no `.map` → `TypeError`.
- **Evidence:** unhandled_rejection on `/`, breadcrumbs `route /` → `click "River"`, app_version
  1782794596298. Same class the create endpoint already patched — see the comment at
  <File path="site/src/routes/api/dictionaries/create/+server.ts" line="113" /> ("`gloss_languages.join(...)`
  threw on the unparsed string").
- **Recommended fix (don't apply):** guard with array-ness, e.g.
  `{#if Array.isArray(dictionary.gloss_languages) && dictionary.gloss_languages.length}`. Cheap,
  removes a homepage crash on the most-visible flow.

### 🟡 orama-watcher delta scan failed — code 21 (SQLITE_MISUSE)
- <File path="site/src/lib/search/orama-watcher.ts" line="72" /> already wraps the scan in try/catch and
  logs at `error` (non-fatal). `context.extras = {"code":21}` = SQLITE_MISUSE — most likely a query
  firing against a wa-sqlite connection mid-teardown (navigation / leadership change). Single
