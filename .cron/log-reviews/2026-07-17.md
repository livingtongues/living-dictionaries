### The `/export` crash — a real P2, one missing optional chain

- **Who / where:** `jwrunner7@gmail.com` (Jacob, L3), **6×** across 3 sessions, all on
  **`/1p-emanuscript/export`**, current build `1784294143202`. Breadcrumbs: `dictionary_opened → /export`
  (crashes on render), and again after `Settings → Export`. Chrome/Linux.
- **Stack:** `TypeError: Cannot read properties of undefined (reading 'glosses')` inside a
  `formatCsvEntries.map` (`prepare-entries-for-csv.ts:90`).
- **Root cause (verified in current source):** `friendly-name.ts:9`
  `let gloss = entry.senses?.[0].glosses` — the `?.` guards `senses` being *null*, **not** `[0]` being
  *undefined*. For an entry that **has audio** (line 90: `entry.audios ? friendlyName(…)`) but whose
  `senses` array is **empty** (`[]`), `entry.senses?.[0]` is `undefined` and `.glosses` throws. The
  `.map` over all entries aborts → the whole export page fails to render.
- **Why it showed up now:** `1p-emanuscript` evidently has ≥1 audio-bearing entry with no sense —
  exactly the shape that trips it. It will hit **any** dictionary with that data shape.
- **Fix (one char, unbuilt, not applied here):** `friendly-name.ts:9` →
  `entry.senses?.[0]?.glosses`. Add a `friendlyName({ senses: [] }, '…wav')` test case. Filed
  `.issues/export-friendlyname-glosses-crash.md`.
