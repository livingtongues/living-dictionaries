*First run of the new SEO/GEO discoverability loop (Loop Library #006, adapted). Read-only audit
against production. No code changed. Establishes the baseline the monthly re-crawl measures against.*

## TL;DR

- **Technical SEO baseline is HEALTHY.** Every priority page-type I crawled emits a real `<title>`,
  a page-specific `<meta name="description">`, a query-stripped `<link rel="canonical">`, and valid
  JSON-LD with the right `@type`. The sitemap index fans out to a per-dict child sitemap, and
  `robots.txt` blocks only `/onondaga/`. Nothing is on fire — this loop's job is to keep it that way
  and to close the **GEO / answer-first** gap, which is where discoverability actually leaks.
- **3 ranked findings**, none a regression (there's no prior run) — all `content/GEO`, not blockers:
  a generic homepage `<title>`, a legacy-id dictionary in the sitemap, and terse/cryptic entry
  descriptions that under-serve the "what does X mean" answer intent.
- **The load-bearing follow-up is building the check into a committed script** so re-observation is
  free — today's pass was inline `curl` + eyeball. Until then the digest is honest but not yet
  cheap to iterate.

## Ranked gaps (by expected impact)

1. **🟡 Answer-first entry descriptions (GEO — highest leverage).** Entry `<meta description>` renders
   as `"n., Spanish: pentágono, L, V, M"` — part-of-speech abbrev + gloss + bare dialect codes
   (`L, V, M`). For the dominant search/AI intent ("what does *pentágonu* mean", "*water* in
   \<language\>") this is cryptic: the answer engine has to guess that `pentágono` is the gloss and
   that `L/V/M` are dialects. A more answer-first shape ("*pentágonu* — pentagon (noun) in A fala")
   would both read better in a SERP snippet and be far more citable by AI answer engines. This is the
   GEO half of the loop and the single most valuable move on the board. *(Source of truth:
   `seo_description.ts` — start the query-benchmark here before touching it.)*
2. **🟡 Generic homepage `<title>` (`Living Dictionaries`).** The most-linked page in the site spends
   its title tag on the bare brand. A subject-bearing title ("Living Dictionaries — build & explore
   dictionaries for endangered languages") would capture more head-term intent without harming brand
   recall. Low-risk, high-visibility. *(Source: `seo-title.ts` / `SeoMetaTags` home usage.)*
3. **🟡 Legacy-id dictionary in the sitemap index.** `/sitemap.xml` lists a child keyed by a raw id
   (`.../sitemaps/80CcDQ4DRyiYSPIWZ9Hy.xml`) alongside slug-keyed children (`a-fala`, `ache`, …). The
   `[dictionaryId]` layout redirects legacy ids to the canonical slug, so this likely resolves — but
   a sitemap that advertises the non-canonical id invites Google to crawl the redirect form and can
   muddy canonical consolidation. Verify the child + its entry URLs emit the **slug** canonical, and
   consider emitting only slug-keyed children. *(Source: `src/routes/sitemap.xml`, `sitemap-helpers.ts`.)*
