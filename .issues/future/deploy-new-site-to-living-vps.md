## Goal

Get the `site` SvelteKit app building into a Docker image and deploying to the living VPS automatically on push, following the same patterns as tutor → poly VPS.

## Current State

- `site/` exists as a workspace package with a bare SvelteKit scaffold (adapter-auto, hello world page)
- `living.conf` already points to this repo (`livingtongues/living-dictionaries`), domain `new.livingdictionaries.app`, deploy branch `svelte-5-migration`, no BUILD_TARGET
- VPS infrastructure (Caddy, webhook, deploy.sh, github-token.sh) is already set up via `vps-setup/bin/sync` — the living VPS just needs a working Dockerfile in this repo
- The living VPS webhook triggers `deploy.sh` which clones the repo, copies `.env`, then runs `docker compose build && docker compose up -d` using `docker-compose.yml` generated by `bin/sync`
- The generated `docker-compose.yml` for living looks at `./code/Dockerfile` (the repo root Dockerfile)

## What Needs to Happen

### 1. Switch adapter-auto → adapter-node in site
- Install `@sveltejs/adapter-node`
- Update `svelte.config.js` to use adapter-node
- Add a `build` script that builds the SvelteKit app

### 2. Create root Dockerfile (modeled on tutor's)
The living VPS `docker-compose.yml` (generated by sync) expects a `Dockerfile` at repo root. It does:
```
build:
  context: ./code
  dockerfile: Dockerfile
```

The Dockerfile needs to:
- **Builder stage**: Install pnpm, copy workspace config + lockfile, copy `site/package.json` and any needed workspace packages, `pnpm install --frozen-lockfile`, copy source, build
- **Runner stage**: Copy build output, install prod deps, run with node

Key differences from tutor:
- Much simpler — no capacitor plugins, no BUILD_TARGET arg, no varlock, no custom server.js, no .env.schema
- Only need to build `site` workspace package
- Also copy any `packages/*` that site depends on (currently none based on package.json)
- Run with plain `node build` since no custom server needed

### 3. Create root docker-compose.yml (for local testing)
Simple compose file like tutor's for local `docker compose up --build` testing.

### 4. Update root package.json build script
Currently `"build": "pnpm --filter=site build"` — need to add a build command for site or change the existing one. Since the old `site` package may still exist in `packages/`, we should add `"build:site": "pnpm --filter=site build"` and have the Dockerfile use that.

### 5. Update living.conf deploy branch (if needed)
Currently set to `svelte-5-migration`. Need to confirm this is correct or if it should be `main`. The webhook only triggers on pushes to the configured branch.

## Implementation Details

### Dockerfile (at repo root)

```dockerfile
FROM node:24-alpine AS builder
WORKDIR /app
RUN corepack enable pnpm

COPY pnpm-lock.yaml pnpm-workspace.yaml package.json ./
COPY site/package.json site/
# Copy any packages/* that site needs (none currently)

RUN pnpm install --frozen-lockfile --ignore-scripts

COPY site/ site/

RUN pnpm --filter=site build

FROM node:24-alpine AS runner
WORKDIR /app

COPY --from=builder /app/site/build build/

ENV NODE_ENV=production
ENV PORT=3000
EXPOSE 3000

CMD ["node", "build"]
```

Note: adapter-node output is self-contained — no need for node_modules in the runner stage (it bundles dependencies). ### Revised runner stage (safer):
```dockerfile
FROM node:24-alpine AS runner
WORKDIR /app
RUN corepack enable pnpm

COPY --from=builder /app/pnpm-lock.yaml /app/pnpm-workspace.yaml /app/package.json ./
COPY --from=builder /app/site/package.json site/
RUN pnpm install --prod --frozen-lockfile --ignore-scripts

COPY --from=builder /app/site/build site/build/

WORKDIR /app/site
ENV NODE_ENV=production
ENV PORT=3000
EXPOSE 3000

CMD ["node", "build"]
```
